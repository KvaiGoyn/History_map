# Историческая карта Санкт-Петербурга

Интерактивная карта с точками интереса, маршрутом и модальным окном с фактами.

## Функциональность

- Отображение карты Санкт-Петербурга с использованием Яндекс.Карт.
- Динамическая загрузка данных о локациях из markdown файлов.
- Отображение меток (Placemark) на карте.
- Построение маршрута (Polyline) между точками в порядке `order`.
- Модальное окно с фотографией и каруселью фактов, открывающееся по клику на метку.
- Карусель реализована с помощью Swiper.
- Закрытие модального окна по крестику, клику вне окна или клавише Escape.
- Индикатор загрузки (loader) во время инициализации.

## Установка и запуск

1. **Получение API-ключа Яндекс.Карт**

   - Перейдите на [страницу разработчика Яндекс.Карт](https://developer.tech.yandex.ru/).
   - Авторизуйтесь с помощью Яндекс ID.
   - Создайте новый ключ API в разделе "Ключи".
   - Выберите тип "JavaScript API и HTTP Геокодер".
   - Ограничьте ключ по HTTP-реферерам (можно указать `localhost:*` для разработки) или оставьте без ограничений.
   - Скопируйте полученный ключ.

   Замените `YOUR_API_KEY` в файле `index.html` (строка 9) на ваш ключ:

   ```html
   <script src="https://api-maps.yandex.ru/2.1/?apikey=ВАШ_КЛЮЧ&lang=ru_RU" type="text/javascript"></script>
   ```

   **Примечание:** Если ключ не указан, карта не загрузится. Для тестирования можно использовать демо-ключ (не гарантируется работа).

2. **Запуск локального сервера**

   Поскольку загрузка markdown файлов использует `fetch`, необходимо запустить проект через HTTP-сервер (протокол `file://` не поддерживает CORS).

   Проще всего использовать встроенный в Python HTTP-сервер:

   ```bash
   python -m http.server 8000
   ```

   Или установите и используйте [Live Server](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer) для VS Code.

3. **Откройте в браузере**

   Перейдите по адресу [http://localhost:8000](http://localhost:8000) (или другому порту, который вы используете).

## Настройка данных

Данные о локациях хранятся в markdown файлах в папке `md/`. Каждый файл должен иметь следующую структуру:

```markdown
---
id: 1
title: "Название локации"
coordinates: [59.9386, 30.2901]
image: "images/имя_файла.jpg"
order: 1
---

### Описание
Текст описания.

### Факты
- Факт 1.
- Факт 2.
```

- `coordinates` — широта и долгота в формате `[широта, долгота]`.
- `image` — путь к изображению относительно корня проекта.
- `order` — порядковый номер для построения маршрута.

## Возможные проблемы и решения

### Карта не отображается
- Проверьте, что ключ API Яндекс.Карт указан верно и активен.
- Убедитесь, что есть интернет-соединение для загрузки API.

### Данные локаций не загружаются
- Запустите проект через HTTP-сервер (не через `file://`).
- Проверьте, что файлы в папке `md/` существуют и имеют корректный формат.

### Изображения не отображаются в модальном окне
- Убедитесь, что пути в `image` указаны правильно и файлы существуют в папке `images/`.
- Проверьте консоль браузера на наличие ошибок загрузки.

### Карусель фактов не листается
- Убедитесь, что подключена библиотека Swiper (подключается через CDN в `index.html`).
- Проверьте, что факты загружены и созданы слайды.

### Как обновить сайт
1. Внесите изменения в код
2. Закоммитьте и запушьте в ветку `master`:
   ```bash
   git add .
   git commit -m "Описание изменений"
   git push origin master
   ```

4. Проверить статус деплоя можно в репозитории на вкладке **Actions**.
